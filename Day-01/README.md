### Agentic AI will:

## 3. Evaluation-Driven Development (EvDD)

### EvDD means:
You let AI generate code or content → then you evaluate, test, and improve it.
This is the new development style for the AI era.
Why EvDD is used:
AI sometimes makes mistakes.

### So developers:
Ask AI to generate code
Evaluate: “Is it correct?”
Give feedback to AI
AI improves the output until perfect

Real-life example:
You ask AI:
“Create a login API.”

AI generates it.
You test it.
If it fails, you say:
“Fix the JWT issue.”
AI fixes it.
This is EvDD: develop by continuously evaluating AI output.

## 4. Test-Driven Development (TDD)

Meaning:
TDD is the old but still important method:
You write tests first → then write the code that passes the tests.
Still used in AI era because:
Tests prevent mistakes
Tests help verify AI-generated code
Tests ensure safe deployments

### Real-life example:

Write a test: “User should not log in with wrong password.”
Ask AI to write the login code.
Run the test.
If test fails --> ask AI to fix code.
